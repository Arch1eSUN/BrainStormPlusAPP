# 0.0 — 当前基础收口：确认 Web Source of Truth，并建立 iOS 1:1 迁移基础盘

**执行模型：Gemini 3.1 Pro**

## 0. 本轮定位

这是 BrainStorm+ iOS App 的第一个正式开发轮次。

本轮不是做全量功能迁移，也不是先做漂亮 UI。当前项目已经有部分 SwiftUI 模块、Supabase 接入、设计系统雏形和 Web 端完整业务系统。现在最重要的是先把后续所有开发不会跑偏的基础盘建立起来。

本轮目标：
1. 明确 Web 端当前真实 source of truth。
2. 明确 App 当前已实现内容与缺口。
3. 收口设计语言来源，避免继续被旧设计文件误导。
4. 建立 iOS 端后续 1:1 迁移必须依赖的基础结构：模块清单、路由映射、设计 token 映射、RBAC 映射、数据同步基础约定。
5. 修复当前 App 里最明显会阻碍后续迁移的占位/硬编码问题，但不做大范围业务模块铺开。

完成本轮后，Winston 会进行审计。审计通过后才进入 `1.0`。审计不通过，则进入 `0.0-0.1` 修复轮次。

## 2. 项目最终目标，必须写入本轮文档

BrainStorm+ iOS App 的最终目标是：

- 与 Web 端共享同一套 Supabase 数据。
- 与 Web 端实现实时同步和互通。
- 与 Web 端保持相同业务规则、RBAC、capability、RLS、安全边界。
- 与 Web 端保持一致的设计语言、材质、配色、字体方向、logo、品牌表达。
- Web 端所有功能最终都要在 iOS 端 1:1 原生重构。
- iOS 端不能只是 Web companion，也不能是功能裁剪版。
- iOS 端需要优秀的原生动效、触感反馈、转场、加载、错误、空状态和可访问性。
