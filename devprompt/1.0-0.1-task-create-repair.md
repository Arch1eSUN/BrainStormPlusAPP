## 3. 修复 B1 — 挂载 CreateTaskView sheet

当前 plus button 设置：

```swift
isShowingCreateTask = true
```

但没有 `.sheet(isPresented:)` 挂载。

必须在 `TaskListView` 的合适 root chain 上添加：

```swift
.sheet(isPresented: $isShowingCreateTask) {
    CreateTaskView(viewModel: viewModel)
}
```

要求：

- 点击 + 必须能打开 sheet。
- 成功 create 后 dismiss。
- 失败时不 dismiss，显示错误。

## 4. 修复 B2 — 对齐 Web/RLS-critical insert payload

Web `createTask` 会写：

```ts
title
description
status
priority
owner_id
assignee_id
project_id
due_date
reporter_id
created_by
progress
```

本轮至少必须让 iOS 在没有 owner picker 时使用当前登录用户作为 owner/reporter/creator：

- `owner_id = current authenticated user id`
- `assignee_id = current authenticated user id`，用于 legacy compatibility
- `reporter_id = current authenticated user id`
- `created_by = current authenticated user id`
- `progress = 0`

这不是伪造数据，而是 Web create action 的 fallback 行为：无 owner 时 creator 是 owner。

实现要求：

1. 在 `TaskListViewModel.createTask(...)` 里获取当前 Supabase authenticated user。
2. 如果无法获取 user id，抛出明确错误：未登录 / 无法创建。
3. `TaskInsert` 加入上述字段。
4. 不使用 service-role key。
5. 不硬编码任何 UUID。
6. 插入成功后 `fetchTasks()`。

## 5. 修复 B3 — due_date 格式

当前不可用：

```swift
ISO8601DateFormatter().string(from: dueDate)
```

必须改为 `yyyy-MM-dd`：

```swift
let formatter = DateFormatter()
formatter.calendar = Calendar(identifier: .gregorian)
formatter.locale = Locale(identifier: "en_US_POSIX")
formatter.timeZone = TimeZone(secondsFromGMT: 0)
formatter.dateFormat = "yyyy-MM-dd"
```

如果选择本地时区也可以，但必须说明原因。默认建议 UTC，减少跨端 off-by-one。
