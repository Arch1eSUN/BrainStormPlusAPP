## 3. 目标

### 目标 A — 建立 Projects list task count 数据层

必须先确认 Web source of truth 对 task count 的真实来源。

允许实现方式按真实 Web source 决定，优先级如下：

1. 如果 Web `fetchProjects()` 直接返回 `task_count` / computed field：
   - iOS 应建立对应 decode DTO / model strategy
   - 不要破坏现有 `Project` flat decode

2. 如果 Web list 页面是通过 `tasks` 数据本地/额外查询计算：
   - iOS 应采用同语义的 batched fetch
   - 必须避免 N+1 per project queries

3. 如果 Web 当前 UI 没有实际显示 task count，但 parity docs/ledger 认定其为 gap：
   - 必须写明 source of truth discrepancy
   - 做最小、可审计 foundation：batched `tasks` count by project ids，供 list card 显示

最低要求：

- 非空 list 时，不得为每个 project 单独发一次 tasks count 查询
- count 失败不得污染 main list fetch error
- count 失败时 list 仍可展示 project rows
- 空 project list 不应发 count 查询

### 目标 B — UI 显示 task count

在 `ProjectCardView` 或 list row 层展示 task count。

最低要求：

- 显示文案清楚，例如：`0 tasks` / `1 task` / `N tasks`
- 不破坏已有 owner byline、avatar、status、progress、end date
- 视觉层级应低于 title/status，高于纯 debug 信息
- 不做大型 card redesign
- 不引入 AI slop 视觉：不要加无意义 gradient metrics / decorative cards

### 目标 C — State truth / failure isolation

不得回退：

- 1.5 list membership scoping
- 1.6 detail membership gate
- 1.7 owner hydrate
- 1.8 avatar/nested name rendering
- 1.9 edit/member foundation
- 2.0 delete foundation

新增 task count state 必须与以下错误面隔离：

- `errorMessage` main list fetch
- `ownersErrorMessage`
- `deleteErrorMessage`
- edit/member error surfaces
- detail enrichment errors

### 目标 D — Scope control

本轮可以做：

- task count data fetch / decode / state map
- list card task count display
- ledger update
- small helper DTO / helper view

本轮不要做：

- AI summary
- risk analysis
- linked risk actions
- resolution feedback
- create/edit/delete/member redesign
- schema changes
- task detail redesign
- task CRUD changes
- batch actions
- analytics dashboard
- large UI redesign

### 目标 E — Ledger truth

更新：

- `findings.md`
- `progress.md`
- `task_plan.md`
- 如有必要新增 `docs/parity/37-winston-ready-2.1-notes.md`

必须明确写：

- task count source of truth 是什么
- iOS 使用何种数据策略
- 是否 batched、是否避免 N+1
- count fetch failure 如何隔离
- 哪些 Projects gap 仍未完成
- `.projects` 是否仍为 `.partial`

默认预期：**`.projects` 仍保持 `.partial`**。
因为 AI summary / risk analysis / linked actions / resolution feedback 仍未完成。
