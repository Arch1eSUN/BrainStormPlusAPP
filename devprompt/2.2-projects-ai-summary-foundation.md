## 3. 目标

### 目标 A — 先核清 Web AI summary 的真实 source of truth

必须先独立确认 Web 上 Projects AI summary 的真实入口与真实数据路径。

至少回答清楚：

1. Web 的 AI summary 是否存在真实按钮 / 面板 / section？
2. 它走 server action、API route，还是直接 client fetch？
3. 输入是什么：project id、detail payload、tasks/daily/weekly 聚合，还是别的？
4. 输出 shape 是什么：纯文本、结构化字段、缓存对象，还是 mixed shape？
5. 失败时 Web 如何表现？

如果 Web 根本没有真实可用 AI summary，只是 placeholder / dead code / vestigial type：
- 必须如实记录 source-of-truth discrepancy
- 不得假装 parity 已存在
- 可以做最小 foundation，但必须在 ledger 里明确写清“Web 当前真实状态”

### 目标 B — iOS Detail AI Summary Foundation

仅在 `ProjectDetailView` 路径上补一个**最小可审计**的 AI summary foundation。

最低要求：

- 有明确入口（按钮或 section 内触发）
- 有 loading state
- 有 success state
- 有 isolated failure state
- 成功后用户能在 detail 页面看到 summary
- 不阻断原有 owner / tasks / daily / weekly detail 内容
- 不做大 UI 改版

foundation 允许的最小形态示例：

- `Generate summary` / `Refresh summary` button
- 点击后请求 Web 对应 summary endpoint / action
- 在 detail 页 read-only 展示一段 summary 文本

### 目标 C — State truth / failure isolation

新增 AI summary state 必须与以下错误面隔离：

- `errorMessage`（detail first-load / gate / project fetch）
- `enrichmentErrors[.tasks|.dailyLogs|.weeklySummaries]`
- `deleteErrorMessage`
- list fetch / owner hydrate / task-count surfaces
- edit/member surfaces

最低要求：

- AI summary 请求失败不得让 detail 页整体报错
- AI summary 失败不得抹掉已加载的 project / tasks / daily / weekly 内容
- AI summary loading 不得阻断 delete / edit / navigation 主路径

### 目标 D — Scope control

本轮可以做：

- Web AI summary source re-read
- iOS detail AI summary request state
- read-only summary UI
- 最小 DTO / response decode
- ledger update

本轮不要做：

- risk analysis
- linked risk actions
- resolution feedback
- schema changes
- long-term persistence / offline cache
- task-count redesign
- create/edit/delete/member redesign
- task CRUD redesign
- AI multi-step workflow orchestration
- streaming chat UI
- prompt engineering backend rewrite
- analytics / telemetry system

### 目标 E — Ledger truth

更新：

- `findings.md`
- `progress.md`
- `task_plan.md`
- 如有必要新增 `docs/parity/39-winston-ready-2.2-notes.md`

必须明确写：

- Web AI summary 的真实 source of truth
- iOS 采用的调用方式
- summary state shape
- failure isolation 方式
- 仍未完成的 Projects gaps
- `.projects` 是否仍为 `.partial`

默认预期：**`.projects` 仍保持 `.partial`**。
因为 risk analysis / linked actions / resolution feedback 仍未完成。
