## 0. 背景

Winston 2.5 独立审计未通过：

- `docs/parity/46-winston-audit-2.5.md`

失败原因只有一个：

> iOS `ProjectResolutionFeedback.governanceSignal` 在 `hasEffective` 与 `needsIntervention` 同时成立时采用了 `.needsIntervention` 优先；但 Web source-of-truth 实际是 `hasEffective` 优先。

本轮是 **最小修复轮**。禁止扩大范围。

## 1. Web source-of-truth

重新读取并确认：

- `BrainStorm+-Web/src/app/dashboard/projects/page.tsx`
- resolution feedback block 中 governance intervention status 的逻辑：

```tsx
const hasEffective = resSummary.recentResolutions?.some(
  r => r.effectiveness === 'effective' && r.category === 'root_cause_fixed'
)
const needsIntervention = resSummary.reopenedCount > 0 && resSummary.active > 0
if (hasEffective) return <span>干预已生效</span>
if (needsIntervention) return <span>待治理干预</span>
return null
```

所以 governance status priority 必须是：

1. `hasEffective`
2. `needsIntervention`
3. none

注意：Web 的 predictive `易重开` badge 是独立 badge，仍可在 `reopenedCount > 0 && active > 0` 时显示。

## 3. 必须修复

### 3.1 修改 `governanceSignal` priority

当前错误逻辑：

```swift
let needs = reopenedCount > 0 && active > 0
if needs { return .needsIntervention }
let effective = recentResolutions.contains { ... }
return effective ? .interventionEffective : .none
```

必须改为：

```swift
let effective = recentResolutions.contains { resolution in
    resolution.effectiveness == "effective" && resolution.category == "root_cause_fixed"
}
if effective { return .interventionEffective }
let needs = reopenedCount > 0 && active > 0
return needs ? .needsIntervention : .none
```

### 3.2 保留 `isProneToReopen`

`isProneToReopen` 仍保持：

```swift
reopenedCount > 0 && active > 0
```

### 3.3 UI 行为

如果同时满足 effective 与 prone-to-reopen：

- governance banner title 应是：`Intervention effective`
- 仍应显示 prone-to-reopen subline / indicator

不要把 prone-to-reopen 与 governance priority 混成同一语义。

### 3.4 注释与 ledger 对账

更新所有错误描述，尤其是：

- 不得再写 “danger trumps success”
- 不得再写 “needsIntervention wins priority when both fire”
- 应改为：
  - Web governance status uses effective-first priority
  - iOS keeps prone-to-reopen as an independent indicator
