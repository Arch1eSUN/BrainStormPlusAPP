## 3. 目标

### 目标 A — 建立 Projects list foundation

实现一个真实可编译的 iOS Projects list 基础页，最小要求：

- `ProjectListView`
- `ProjectListViewModel`
- 使用现有 Supabase client
- 从 `projects` 表读取项目列表
- loading / empty / error state
- list row 至少展示：项目名称 / 状态或基础元信息 / 可选描述或日期
- UI 采用现有 iOS app design tokens / component style

如果已有 Projects 相关模型或 view model，优先复用，不要重复造类型。

### 目标 B — Wire typed routing

将：

```swift
AppModule.projects
```

从 `ParityBacklogDestination` 接到真实 `ProjectListView`。

同时重新评估：

```swift
AppModule.implementationStatus
```

如果 Projects list foundation 已真实可用，应从 `.partial` 调整为更准确状态；但不要标成 full parity unless justified。可保留 `.partial` 并在 ledger 中说明：list foundation exists but full project CRUD/details parity pending。

### 目标 C — Ledger truth

- `findings.md`

- 如有必要新增 `docs/parity/21-winston-ready-1.3-notes.md`
