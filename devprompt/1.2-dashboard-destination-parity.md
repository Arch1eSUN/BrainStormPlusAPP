## 3. 目标

### 目标 A — 覆盖审计

逐项核对 `AppModule` 与 `ActionItemHelper.destination(for module:)`：

- 哪些模块已经有真实 destination
- 哪些模块仍落到 `ParityBacklogDestination`
- 哪些模块只是壳页面/占位页面但已不是 backlog destination

必须给出源码级清单，不接受口头自报。

### 目标 B — 修复一批高价值 destination parity 缺口

优先级原则：

1. Dashboard 主路径直接可点击
2. 有现成 view / 容器可接，不需要发散到整套新领域实现
3. 可独立 build 验证
4. 不破坏 typed routing

如果某模块没有足够实现基础，不要硬做假页面；应在 ledger 中准确记录为 deferred debt。
